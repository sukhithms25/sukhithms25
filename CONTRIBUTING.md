# Contributing Guidelines

Thank you for your interest in contributing! Please review the guidelines below to ensure a smooth collaboration.

## Code Quality Standards
- All Python code must be formatted with **Black** and linted with **Ruff**.
- Run **mypy** for static type checks before committing.
- Write tests for any new features or bug fixes.

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Implement your changes and write unit tests.
3. Ensure CI checks pass locally.
4. Submit a PR describing your summary, changes, testing, and checklist.
