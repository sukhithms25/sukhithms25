# Readme Layout & Visual Hierarchy Research

A successful layout guides the user's eye from top to bottom. We researched the layout structure of top developers:

## Visual Hierarchy Rules
1. **The F-Shape Pattern:** Visitors read the top header, scan the next line, and then scroll vertically, glancing at headings.
2. **Whitespace:** At least 20-30% of the vertical space should be white space (or empty dark space) to let the page "breathe."
3. **Contrast:** Key values (like language names or project statuses) should stand out using bold tags or custom colors, while details should be muted.
